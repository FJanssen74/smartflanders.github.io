Slimme opmaak tutorials

Dit document is kladversie 0.1

Steeds meer organisaties wensen hun data te ontsluiten volgens de principes van Linked Data. Hun “zaken”, bijvoorbeeld een feed van nieuwsberichten, worden vaak ontsloten met hun website, maar is niet leesbaar voor machines. Eén methode om de data te publiceren op het Web is om deze webpagina’s te annoteren met Linked Data (cfr. slimme opmaak). Doordat deze methode meer gefragmenteerd is dan traditionele publicatiemethodes (SPARQL-endpoint of RDF data dump) is er onzekerheid over hoe een eindgebruiker automatisch de data van de ganse website kan verzamelen en vragen hierover kan stellen. Met behulp van tutorials tonen we hoe een eindgebruiker zelf aan de slag kan met geannoteerde data. Dit document beschrijft hoe een spreadsheet (CSV-bestand) gegenereerd kan worden en hoe vragen (SPARQL-queries) gesteld kunnen worden, zowel over de website-data als over andere databronnen zoals Wikidata. Door deze tutorials te volgen en de gebruikte dataset te analyseren krijgen organisaties niet alleen meer inzicht hoe slimme opmaak een kosten-efficiënte manier van Linked Data publicatie is, maar kunnen ook controlleren of de data bevraagbaar is met de aangeboden toolset. De dataset van deze tutorials maakt enkel gebruik van paginering om de subject-pages aan elkaar te linken. Als toekomstig werk worden er andere vormen van hypermedia toegevoegd om bijvoorbeeld full-text-search over basislijsten te kunnen doen.

Gebruikte dataset

De tutorials maken gebruik van een recent gepubliceerde slimme-opmaak-dataset: het krantenarchief van de eerste Wereldoorlog. Deze website bevat een gepagineerde lijst van meer dan 50.000 kranten. De principes van een gepagineerde lijst die hier worden aangekaart voor kranten blijven hetzelfde bij een ander soort objecten. Hierbij zijn er twee zaken waar op gelet moet worden tijdens het opstellen van de Linked Data: eerst moet ervoor gezorgd worden dat de domein-specifieke metadata, hier krantenmetadata, wordt voorzien, ten tweede mag een machine niet enkel data van één pagina terugvinden, maar ook kan verder zoeken naar de volgende of vorige pagina’s met behulp van hypermedia links.

In het voorbeeld van kranten kan je bovenstaande zaken bekijken door op ‘Bekijk paginabron’ (View page-source) te klikken bij een krant. Bovenaan zie je een script-tag met de Linked Data in JSON-LD formaat, de zogenaamde rich snippet. De domein-specifieke metadata voor een krantenpagina wordt als volgt weergegeven:

Tutorial 1: uw website als een spreadsheet

Linked Data gebruikt het Resource Description Framework (RDF) als methode om kennis weer te geven. Elk stukje kennis wordt voorgesteld als een triple: subject - predikaat - object. Deze stukjes samen kunnen weergegeven worden als een graaf waardoor RDF dus een graafmodel is. Bij een spreadsheet wordt een tabulair formaat gebruikt, bijvoorbeeld het Comma Seperated Value (CSV) formaat. De graaf van triples zal bijgevolg omgevormd worden naar een tabel zoals weergegeven in onderstaande figuur. Het idee hierbij is dat in de meest linkse kolom het onderwerp (subject) staat waarover uitgespraken gedaan worden. De kolomwaardes in de header stellen de predikaten voor en de cellen per rij stellen de objects voor. Deze objects kunnen ofwel verwijzen naar een ander onderwerp (complex object) of bevatten een echte waarde (string, integer, datum...).

See the Pen Slimme opmaak als spreadsheet by Brecht Van de Vyvere (@brechtvdv) on CodePen.

Deze manier van voorstellen zorgt er weliswaar voor dat verbanden tussen complexe objecten moeilijker te benaderen zijn in spreadsheets. De spreadsheet is bijgevolg meer een verkennende methode voor niet-technische profielen. In volgende tutorial wordt getoond hoe rechtstreeks vragen gesteld kunnen worden over de Linked Data.
